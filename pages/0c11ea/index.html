# createContext (opens new window)

createContext lets you create a context that components can provide or read.

让我们通过一个简单的示例来说明如何使用上下文对象在 React 应用中进行全局状态管理。

假设我们有一个简单的计数器应用, 其中包含一个按钮组件和一个显示计数的组件。我们希望能够在应用的任何地方访问和更新计数器的值, 而不需要通过 props 一层一层地传递。

首先, 我们可以创建一个上下文对象来存储计数器的值和更新函数:

// CounterContext.js
import React, { createContext, useState, useContext } from 'react';

const CounterContext = createContext();

export const CounterProvider = ({ children }) => {
  const [count, setCount] = useState(0);

  const increment = () => setCount(count + 1);
  const decrement = () => setCount(count - 1);

  return (
    <CounterContext.Provider value={{ count, increment, decrement }}>
      {children}
    </CounterContext.Provider>
  );
};

export const useCounter = () => useContext(CounterContext);

在上面的代码中, 我们创建了一个 CounterProvider 组件, 它使用 useState 来创建一个计数器状态 count, 并提供了增加和减少计数器值的函数 increment 和 decrement。然后, 我们通过 CounterContext.Provider 将这些数据和方法提供给子组件。

接下来, 我们可以使用 useCounter 钩子来访问上下文中的数据和方法:

// Button.js
import React from 'react';
import { useCounter } from './CounterContext';

const Button = () => {
  const { increment, decrement } = useCounter();

  return (
    <div>
      <button onClick={increment}>Increment</button>
      <button onClick={decrement}>Decrement</button>
    </div>
  );
};

export default Button;

最后, 我们在应用的根组件中使用 CounterProvider 包裹整个应用:

// App.js
import React from 'react';
import Button from './Button';
import Display from './Display';
import { CounterProvider } from './CounterContext';

const App = () => {
  return (
    <CounterProvider>
      <Button />
      <Display />
    </CounterProvider>
  );
};

export default App;
