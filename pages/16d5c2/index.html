# ref 为null的情形

在 React 中，ref.current 为 null 通常有以下几种常见原因:

  1. 组件尚未挂载
  • 如果你在组件挂载之前访问 ref.current，它会是 null。React 在组件挂载后才会为 ref.current 赋值
  1. 元素或组件没有正确关联到 ref
  • 如果 ref 没有正确地传递给 DOM 元素或组件，ref.current 将保持 null。确保你已经正确地将 ref 赋值给元素或组件。
const inputRef = React.useRef(null);

React.useEffect(() => {
    console.log(inputRef.current); // 如果正确挂载，这里应该不是null
}, []);

return <input ref={inputRef} />;

  1. 条件渲染导致元素未渲染
  • 如果元素因为条件渲染没有被渲染到页面上，ref.current 会是 null。
const inputRef = React.useRef(null);
const [showInput, setShowInput] = React.useState(false);

React.useEffect(() => {
    console.log(inputRef.current); // 由于input元素未渲染，这里会是null
}, []);

return showInput && <input ref={inputRef} />;

  1. 自定义组件未正确转发 ref
  • 如果你将 ref 传递给自定义组件，而该组件没有使用 React.forwardRef 来转发 ref，那么 ref.current 会是 null。
const MyInput = React.forwardRef((props, ref) => {
    return <input ref={ref} {...props} />;
});

const inputRef = React.useRef(null);

return <MyInput ref={inputRef} />;

  • 函数组件本身不会自动处理 ref，你需要使用 React.forwardRef 来转发 ref。

# 父组件传递状态到子组件

总体来说，整理了三个方法:

  1. 通过ref转发,(dom未挂载场景下，ref的current为null)
  2. 通过props传递状态

# 通过props传递状态

父组件

import React, { useState } from 'react';
import ChildComponent from './ChildComponent';

function ParentComponent() {
    const [isActive, setIsActive] = useState(false);

    // 模拟父组件状态的变化
    const toggleActiveState = () => {
        setIsActive(!isActive);
    };

    return (
        <div>
            <button onClick={toggleActiveState}>
                {isActive ? 'Deactivate' : 'Activate'}
            </button>
            <ChildComponent isActive={isActive} />
        </div>
    );
}

export default ParentComponent;

import React, { useEffect } from 'react';

function ChildComponent({ isActive }) {
    useEffect(() => {
        if (isActive) {
            console.log('Child Component is active!');
        } else {
            console.log('Child Component is inactive.');
        }
    }, [isActive]);

    return (
        <div>
            {isActive ? 'The child is active' : 'The child is inactive'}
        </div>
    );
}

export default ChildComponent;

代码解释

  • ParentComponent (父组件):

    • 维护一个状态 isActive，并通过按钮点击来切换状态。将 isActive 作为 prop 传递给 ChildComponent。

  • ChildComponent (子组件):

    • 接收 isActive 作为 prop，并在 useEffect 中监听 isActive 的变化。每次 isActive 变化时，useEffect 都会触发，并输出相应的信息。
如何工作

状态切换: 当你在父组件中点击按钮时，isActive 状态切换，子组件会接收到新的 prop 值并触发 useEffect。 挂载后的处理: 即使子组件在父组件状态变化时尚未挂载，子组件挂载后依然会接收到当前的 isActive 值，并根据最新的值执行逻辑。 通过这种方式，父组件可以在子组件挂载前就通知其状态，而子组件在挂载后会自动根据接收到的 prop 执行相应的操作。

  • 父组件在合适的时候通过 tabContext 直接调用目标方法
子组件 
export const CommonTabs = (props: CommonTabsProps) => {

            releaseSubPage()
        }
        props.tabContext.setPage = (page: number, smoothScroll?: boolean) => {
            setPage(page, smoothScroll)
        }
    }

    return (
      // jsx
    )
}
