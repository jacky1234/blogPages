# 常用命令

提示

此模块列举了常见的一些命令, 更多的用法可见:链接 (opens new window)

  • docker run: 运行一个容器(这个命令也创建了一个容器)。

    • -d: 表示以后台(守护进程)模式运行容器。容器将在后台运行, 而不会占用你的终端
    • -p: 指定端口映射。
    • --name: 指定容器名称
    • --user root: 以 root 模式运行容器
    • --user 1000:1000: 其中 1000:1000 是一个具有普通权限的用户和用户组的标识。

  • docker build: 构建一个镜像

    • -t: 指定镜像的名称和标签
    • -f: 指定 Dockerfile 的路径

  • docker pull <image:version>: 下载一个镜像.

    • docker pull elasticsearch:<version> 下载 elasticsearch 镜像
    • docker image pull elasticsearch

  • docker push: 将本地镜像推送到镜像仓库

  • docker stop <container>: 停止一个正在运行的容器

  • docker start <container>: 运行一个容器

  • docker restart <container>: 重启一个容器

  • docker rm: 删除一个容器

    • -f: 强制删除正在运行的容器

  • docker rmi: 删除一个镜像

    • -f: 强制删除正在使用的镜像

  • docker ps: 查看正在运行的容器

    • -a: 查看所有容器, 包括已停止的

  • docker images: 查看本地镜像列表

  • docker exec: 在正在运行的容器中执行命令

    • docker exec -u root -it <container> /bin/bash: 以 root 进入 Docker 容器的 shell
    • -u root: 你可能需要以此身份运行 docker container, 否则有些命令无权限

  • docker logs <conatiner-name>: 查看容器的日志输出

  • docker inspect: 检查容器或镜像的详细信息

  • docker export: Export a container’s filesystem as a tar archive

  • docker image prune 清理无用镜像,释放磁盘空间

  • docker container: container help

    • docker container prune: 清理无用的容器,释放磁盘空间
    • docker container ls -a

  • docker cp

    • docker cp <path> <container>:<path>: 将主机的文件拷贝到 docker 容器中

# 技巧

为 Docker for Mac 配置 HTTP 代理: https://zhuanlan.zhihu.com/p/26033249 (opens new window)

  • 使用 Dockerfile: 编写一个 Dockerfile, 定义容器的构建过程, 并使用 docker build 命令构建镜像。
  • 镜像标签管理: 给镜像打上标签, 可以方便地区分不同版本或不同环境的镜像。
  • 数据卷挂载: 使用数据卷挂载容器内的目录到主机上, 实现数据持久化, 并方便对数据进行备份和恢复。
  • 使用 Docker Compose: 使用 Docker Compose 工具, 可以通过一个 YAML 文件定义和管理多个相关容器的配置, 方便地进行多容器应用的部署和管理。
  • 多阶段构建: 在 Dockerfile 中使用多个构建阶段, 可以减小最终镜像的体积, 提高构建速度。
  • 使用官方镜像: 尽可能使用官方提供的镜像, 它们经过了广泛测试和验证, 可信度较高。
  • 容器互联: 通过设置网络连接, 可以实现容器之间的通信和数据共享。
  • Docker Registry: 搭建私有的 Docker Registry, 可以方便地存储和分享自己的镜像。
  • 监控和日志: 使用监控工具和日志管理工具, 对 Docker 容器和集群进行监控和日志收集, 方便故障排查和性能优化。
  • 自动化构建和部署: 结合持续集成和持续部署工具

# 原理

以下是 Docker 的工作原理的关键要点:

  • 容器化技术: Docker 使用容器化技术来封装应用程序和其依赖项, 以便它们可以在不同的环境中运行一致, 而无需考虑底层操作系统的差异。这是通过使用 Linux 容器技术(如 cgroups 和 namespace)来实现的。

  • 镜像: Docker 镜像是容器的构建模块。镜像是一个只读文件, 包含了应用程序代码、运行时环境、依赖项等。镜像是不可更改的, 这意味着一旦创建, 它们就不会被更改。镜像是容器的基础, 可以用来启动多个容器实例。

  • 容器: 容器是从镜像创建的运行时实体。容器包括一个独立的文件系统、运行时环境和应用程序代码。容器是可移植的, 可以在不同的主机上运行, 只要这些主机支持 Docker。

  • Docker Daemon: Docker 守护进程(Docker Daemon)是一个在后台运行的服务, 负责管理 Docker 容器的生命周期、镜像的构建和存储、网络设置等。Docker 守护进程监听 Docker CLI 的命令, 并执行相应的操作。

  • Docker CLI: Docker 命令行界面(Docker CLI)是与 Docker 守护进程通信的主要方式。用户可以使用 Docker CLI 来创建、管理和监视容器、镜像以及其他 Docker 资源。

  • Docker Compose: Docker Compose 是一个用于定义和运行多个 Docker 容器的工具。它允许你使用一个 YAML 文件来描述应用程序的各个组件和它们之间的关系, 然后可以一键启动整个应用程序栈。

  • Docker Registry: Docker Registry 是用于存储和分享 Docker 镜像的中央存储库。Docker Hub 是 Docker 的官方 Registry, 用户可以在其中找到许多常见的公共镜像。企业也可以搭建自己的私有 Docker Registry 来管理和分享镜像。

Docker 的工作原理可以简单总结为: 通过将应用程序及其依赖项打包到容器中, 使其可移植性极高, 然后使用 Docker 守护进程管理容器的生命周期和资源。这种容器化方法大大简化了应用程序的部署和管理, 提高了开发、测试和生产环境之间的一致性。
