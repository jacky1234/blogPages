adb

Android Debug Bridge (adb) is a versatile command-line tool that lets you communicate with a device. The adb command facilitates a variety of device actions, such as installing and debugging apps. adb provides access to a Unix shell that you can use to run a variety of commands on a device.

关于 adb 的更多介绍可见: 链接 (opens new window)

  • awesome-adb (opens new window)
  • 官网 (opens new window)

# common

  • adb shell ip address show wlan0 查看 wlan0 网络接口的 IP 地址信息
  • adb shell ifconfig | grep "inet " 获取 Android 设备的 IP 地址
  • adb shell "su -c ifconfig" 查看和配置网络接口信息的命令
  • adb bugreport crash.txt 生成一个完整的设备状态报告, 并包括崩溃日志。它将崩溃日志保存到名为"crash.txt"的文件中
  • adb kill-server 结束 adb 进程
  • adb start-server 启动 adb 进程
  • adb install -r <apk>
  • adb uninstall <package_name>
  • adb reboot
  • adb shell run-as <package> cat <私有路径文件> <输出到mac端的路径>: 拉取应用私有目录到 pc 端, 需要应用 debugable
  • adb shell getprop: list
    • adb shell getprop -T: Show property types instead of values
    • adb shell getprop -Z: Show property contexts instead of values
    • adb shell getprop ro.product.cpu.abi 获取手机支持的 arm 架构
    • adb shell getprop ro.build.version.release 系统版本
    • adb shell getprop | grep -E 'ro.product.model|ro.product.brand|ro.product.device': 获取手机型号

# dumpsys

dumpsys 同来用来 dumpsys 一些内存数据, 通过如下命令可以知道可以 dumpsys 哪些数据 adb shell dumpsys -l

常用的一些 dumpsys 相关的命令有:

  • adb shell dumpsys activity activities 查看前台 Activity
  • adb shell dumpsys activity top
  • adb shell dumpsys activity <activity_name> 获取当前 Activity 中包含的 Fragment 信息, 如 adb shell dumpsys activity com.example.app/.MainActivity, 如下图:
  • adb shell dumpsys activity broadcasts | grep <packagename>: 显示系统中所有广播接收器的信息，包括静态注册和动态注册的广播接收器, 通过 grep 过滤
  • adb shell dumpsys activity intents
  • adb shell dumpsys activity permissions
  • adb shell dumpsys activity processes
  • adb shell dumpsys activity providers
  • adb shell dumpsys activity recents
  • adb shell dumpsys activity services [<package-name>] 查看正在运行的 Services
  • adb shell dumpsys meminfo <进程名称或进程ID> 内存信息
  • adb shell dumpsys window | grep mCurrentFocus 获取设备的当前前台 Activity
  • adb shell dumpsys SurfaceFlinger –latency 用于获取 Android 设备上 SurfaceFlinger 的延迟信息
  • adb shell dumpsys gfxinfo $processName 用于获取特定应用程序进程的图形渲染信息

# am

adb shell am -h

  • adb shell am crash
  • adb shell am kill: 在安全模式下杀死进程, 不影响用户体验
  • adb shell am kill-all: 杀掉所有后台程序
  • adb shell am force-stop: 应用在前台强制关闭, 强制关闭指定的 package 包应用
  • adb shell am start -D
  • adb shell am start -W
  • adb shell am set-debug-app -w $package
  • adb shell am set-debug-app -w --persistent $package
  • adb shell am clear-debug-app $package cancel debug with launch
  • adb shell am broadcast -a $action
  • adb shell am broadcast -a $action -e x.command activity -e name jacky 通过 broadcast 传递参数

# pm

  • adb shell pm list packages
  • adb shell pm list packages -f -3 | grep <too> | sed 's/^package:\(.*\)=.*/\1/': get the path of apk
    • with the -f flag to get the path of the app too
  • adb shell pm path {package}: determine its installed path on the device
  • adb pull /data/app/com.ubercab-1/base.apk com.ubercab.apk: download the APK from the device
  • adb shell pm clear <package>: 清除应用缓存

# ps

adb shell ps 命令提供了一些选项, 可以用来过滤和显示特定的进程信息。以下是一些常用的选项:

  • -A: 显示所有进程, 包括系统进程和应用程序进程。
  • -a: 显示除了无效的进程(已经被 init 启动的进程)之外的所有进程。
  • -x: 显示所有进程, 包括未运行的进程。
  • -r: 按照内存使用量来排序进程。
  • -n: 按照进程名字来排序进程。
  • -s: 按照进程状态来排序进程。
  • -p: 显示进程的 PID(进程标识符)。
  • -c: 显示进程的 CPU 利用率。
  • -u: 显示进程的用户名称和 UID(用户标识符)。
  • -t: 显示进程的线程信息。
  • -Z: 显示进程的安全上下文。

您可以根据需要选择适当的选项来获取所需的进程信息。例如, 使用命令 adb shell ps -p 可以显示进程的 PID, 而使用命令 adb shell ps -t 可以显示进程的线程信息

# logcat

  • adb logcat -h
  • adb logcat -G 16M 将缓冲区大小设置为 16MB
  • adb logcat -c 清除设备日志缓存的 ADB 命令
  • adb logcat -b crash 查看设备上的崩溃日志
  • 日志过滤
    • adb logcat *:V
    • adb logcat *:E
    • adb logcat -s "keyword"
    • adb logcat *:E -s "keyword"
    • adb logcat | grep -E 'regex'

# input

adb shell input 是用于模拟输入事件的 ADB 命令, 它可以在 Android 设备上执行各种输入操作, 如按键、点击、滑动等。

以下是 adb shell input 命令的一些常见用法:

  • 模拟点击事件: adb shell input tap x y 这里的 x 和 y 分别是屏幕上的横纵坐标, 用于指定点击的位置。

  • 模拟按键事件: adb shell input keyevent KEYCODE 这里的 KEYCODE 是表示按键码的整数值, 用于模拟按下指定的按键。(KeyCode 查询地址 (opens new window))

  • 模拟滑动事件: adb shell input swipe x1 y1 x2 y2 [duration] 这里的(x1, y1)和(x2, y2)是滑动起始点和终点的坐标, duration 表示滑动的持续时间。

  • 模拟输入文本: adb shell input text "your_text" 这里的 your_text 是要输入的文本内容, 可以是任意字符串。

  • 长按某个按键: adb shell input keyevent --longpress KEYCODE 这里的 KEYCODE 是表示按键码的整数值, 用于模拟长按指定的按键。

注意: 使用 adb shell input 命令需要连接到 Android 设备, 并且设备必须具有开发者选项中的"USB 调试"选项已启用。此外, 某些命令可能需要 Root 权限才能执行。

# 接口转发

【背景】咨询一个网络的问题, 我这里有两台设备,分别为 PC 和 Android A 设备,这两个设备处于同一个 wifi 下面, 比如 wifi 名称为 wifiA。 另外有一个设备 Android B, 处于一些原因没有连接到 wifiA。而是通过热点连接到 Android A 的热点上, 从而 PC 和 Android B 也能互通私有的资源。 Android B 的 ip 地址为: 192.168.254.193。 另外 Android B 上面安装了 frida 服务, 如果 PC 要访问 Android B 的服务, 我这里通过如下命令进行连接: objection -N --host 192.168.254.193 --port 1234 explore

最终的结果是:

Unable to connect to the frida server: unable to connect to remote frida-server: Could not connect to 192.168.254.193: Operation timed out
1

以下的回答供参考 chatGPT, 并没有测试通过:

根据您提供的信息, 我们可以看到 Android B 连接到了 Android A 的热点上, 这会导致 Android B 和 PC 处于不同的子网中, 因此 PC 无法直接访问 Android B 的 IP 地址。为了解决这个问题, 您可以尝试通过以下步骤进行操作:

一、在 Android A 上启用 IP 转发功能。 在 Android A 的终端上运行以下命令:

su
echo 1 > /proc/sys/net/ipv4/ip_forward
1
2

这将启用 Android A 上的 IP 转发功能, 使其可以将来自 PC 的请求转发到 Android B。

二、设置 Android B 的默认网关。 在 Android B 的终端上运行以下命令:

su
route add default gw 192.168.43.1
1
2

其中, 192.168.43.1 是 Android A 的 IP 地址, 这将设置 Android B 的默认网关为 Android A 的 IP 地址。

三、在 PC 上设置路由规则。 在 PC 上运行以下命令: route add -net 192.168.254.0 netmask 255.255.255.0 gw 192.168.43.119 其中, 192.168.43.119 是 Android A 的 IP 地址, 这将设置 PC 的路由规则, 使其可以将请求发送到 Android A。

# 写入数据到手机，脚本

#!/bin/bash

if [ $# -ne 1 ]; then
  echo "Usage: $0 <input_text>"
  exit 1
fi

input_text=$1
local_temp_file="/tmp/temp_cache_123.txt"

# 检测设备连接情况
connected_devices=$(adb devices | grep -E "device$" | awk '{print $1}')

if [ -z "$connected_devices" ]; then
  echo "没有找到设备，请检查设备连接"
  exit 1
fi

# 如果连接了多个设备，要求用户选择
if [ $(echo "$connected_devices" | wc -l) -gt 1 ]; then
  echo "检测到多个设备，请选择一个进行写入:"
  select device in $connected_devices; do
    if [ -n "$device" ]; then
      echo "选择的设备: $device"
      break
    else
      echo "无效选择，请重试."
    fi
  done
else
  # 只有一个设备
  device=$(echo "$connected_devices" | head -n 1)
  echo "检测到一个设备: $device"
fi

# 将输入文本写入本地临时文件
echo "$input_text" > "$local_temp_file"

# 将本地临时文件推送到指定设备
adb -s "$device" push "$local_temp_file" /sdcard/widget/cache.txt
push_status=$?

# 删除本地临时文件
rm "$local_temp_file"

if [ $push_status -ne 0 ]; then
  echo "写入失败"
else
  echo "写入成功，地址: /sdcard/widget/cache.txt"
fi
