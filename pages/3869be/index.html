# npx

npx 是 npm v5.2.0 及更高版本中附带的一个工具。它用于执行一次性的命令行工具或安装的包。相比于全局安装包并在命令行中调用，npx 可以在不安装全局包的情况下直接运行包。当你只需要临时使用一个工具或者尝试一个新的包时，npx 是一个很方便的选择。

常用 npx 命令

  • npx <command>: 运行命令行工具或包，如果本地没有安装，则会自动下载和运行。
  • npx create-react-app my-app: 创建一个新的 React 应用程序。
  • npx -p <package-name> <command>: 在运行命令之前，安装指定的包。
  • npx -c <command>: 在命令行中执行 JavaScript 代码片段。

# vite

Vite 是一种现代化的前端构建工具，由尤雨溪（Vue.js 的作者）及其团队开发和维护。名称 "Vite" 来自于法语单词，意为 "快速" 或 "迅速"。Vite 的主要目标是提供快速的开发和构建体验，特别适用于 Vue 3 项目，但也可用于其他前端框架和库。

以下是 Vite 的一些主要特点和优势：

  • 快速的开发服务器: Vite 使用原生 ES 模块导入，充分利用现代浏览器的特性，实现了快速的开发服务器，几乎无需等待时间。在开发模式下，它可以在浏览器中即时加载并热更新代码。

  • 快速的冷启动: Vite 的冷启动速度非常快，因为它只编译需要的代码，而不是整个应用程序。这意味着开发者可以快速启动项目并立即开始开发。

  • 按需编译: Vite 会将应用程序的源代码分割成更小的模块，只在需要时进行编译。这种按需编译的方式显著减少了构建时间和资源消耗。

  • 多框架支持: 虽然最初是为 Vue 3 项目设计的，但 Vite 也支持其他前端框架和库，如 React、Preact 等。

  • 简单的配置: Vite 的配置文件非常简单，不需要复杂的配置即可启动项目。这减少了学习曲线，使新手更容易上手。

  • 生产构建: Vite 提供了用于生成生产构建的工具，以优化和压缩代码以用于生产环境。

  • 插件系统: Vite 支持插件系统，可以通过插件来扩展其功能，从构建优化到开发工具等。

总之，Vite 为前端开发提供了更快的开发和构建体验，使开发者能够更高效地构建现代 Web 应用程序。如果你寻求快速、轻量级和开发者友好的前端构建工具，Vite 是一个很好的选择。

# gulp (opens new window)

  • 任务型构建工具: Gulp 是一个任务运行器，你需要编写一系列的任务来执行不同的构建步骤，如编译、压缩、拷贝文件等。这些任务通常使用插件完成，如 gulp-sass 用于编译 Sass，gulp-uglify 用于压缩 JavaScript。

  • 配置驱动: Gulp 是配置驱动的，你需要创建一个 gulpfile.js 文件来定义你的任务。这意味着你可以自定义任务以适应你的项目需求。

  • 插件生态系统: Gulp 拥有庞大的插件生态系统，你可以找到适用于各种任务的插件。这为你提供了极大的灵活性，但也需要更多的配置和设置。

  • 构建过程: Gulp 构建通常是串行的，它会按照你定义的任务顺序执行。

# pnpm vs npm

npm 和 pnpm 都是用于管理 JavaScript 包依赖的工具，主要用于 Node.js 和前端开发。它们有一些共同点，但也有一些区别：

npm (Node Package Manager):

  • npm 是 Node.js 的官方包管理工具，是默认的包管理器，随着 Node.js 一起安装。
  • npm 使用一个本地的 node_modules 目录来存储项目的依赖。
  • 依赖包的版本信息通常存储在 package.json 文件中。
  • npm 提供了大量的包，是 JavaScript 社区的主要生态系统之一。
  • 它可以通过命令行进行安装、更新和卸载依赖包。

pnpm:

  • pnpm 是一种替代性的包管理工具，旨在解决传统 npm 安装依赖时的一些问题，如磁盘空间占用、安装速度和缓存问题。
  • pnpm 采用了一种不同的方式来管理依赖，它使用一个全局的存储库，将依赖包符号链接到项目中，而不是在每个项目中复制依赖包。
  • 这种符号链接的方式可以节省磁盘空间，减少重复的下载，提高安装速度。
  • pnpm 支持 pnpm install、pnpm update 等命令来安装和更新依赖。
  • pnpm 通过一个名为 pnpm-lock.yaml 的文件来记录依赖包的版本信息。

在选择使用 npm 还是 pnpm 时，可以根据项目需求和个人偏好来决定。如果你关注磁盘空间和安装速度，pnpm 可能是一个不错的选择。但请注意，对于一些老旧的项目和工具，可能需要对 pnpm 兼容性进行调整。如果你在一个团队中工作，最好与团队成员一起确定使用哪个工具，以确保一致性。
