react snipB

# 背景

这是我在平时的开发中遇到的一个场景, 具体如下: 假设我需要在 A 组件调用 B 组件的一个方法, 此方法之前是没有参数传递的。现在我需要传入一个回调函数, 用于知道 B 组件当前是否可见。下面是精简的初始代码

// @ts-ignore
import * as React from 'react';
export class B extends React.Component {
	static show:
		| ((isVisibleCallback?: (isVisible: boolean) => void) => void)
		| undefined = undefined;

	showStateCallback: ((isVisible: boolean) => void) | undefined = undefined;

	constructor(props: any) {
		super(props);
		// this.initState();
		B.show = this.show.bind(this);
	}

	show(showStateCallback?: (isVisible: boolean) => void) {
		// 注入当前的 callback
		this.showStateCallback = showStateCallback;
		// @ts-ignore
		this.setState({ show: true, bottom: 100 });
		// ...
	}

	close() {
		// @ts-ignore
		this.setState({ show: false });
	}

	componentDidUpdate(prevProps: any, prevState: any) {
		// @ts-ignore
		if (this.state.show !== prevState.show) {
			// show state changed and callback
			// @ts-ignore
			this.showStateCallback && this.showStateCallback(this.state.show);
		}
	}

	render() {
		// @ts-ignore
		return <h1>Hello, {this.props.name}!</h1>;
	}
}

# 剖析

# 设计 show 方法

希望 show 可以回调到 panel 的状态 (弹起/关闭), 下面是我在 B 组件将 show 方法改成如下格式(这里注释了)

// static show: ((isVisibleCallback?: (isVisible: boolean) => void) => void) | undefined = undefined;
1

为什么有两个 ==>

使用两个 => void 表示两个函数类型的返回值都是 void。

  • 第一个 => void 是用于指定 isVisibleCallback 这个回调函数的返回值为 void
  • 第二个 => void 是用于指定 show 方法本身的返回值为 void。

这样定义的 show 方法可以接受一个可选的回调函数 isVisibleCallback, 这个回调函数接受一个布尔值参数 isVisible 并且没有返回值。同时, show 方法本身也没有返回值。

# 调用

在 A 组件调用方式如下:

B.show?.((isVisible: boolean) => {
	setXXXVisible(isVisible ? 2 : 1);
});

这样我们 A 组件就可以见听到 B 组件 panel 的状态了

# 总结

此问题并不是复杂, 这里记录下解体思路。 另外 B 中 show 方法的定义确实也不熟悉, 摸索成功后果断记录。 回头想其实挺简单的
