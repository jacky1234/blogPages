# command

在正常模式中, 按下:(冒号)键或者 /(斜杠), 会进入命令模式。在命令模式中可以执行一些输入并执行一些 VIM 或插件提供的指令, 就像在 shell 里一样。这些指令包括设置环境、文件操作、调用某个功能等等。

  • set nu 显示行
  • set nonu
  • set nu!: 在行号与非显示行号切换
  • :n: 定位指定行
  • :set ic: 编辑器将不会区分大小写
  • :set noic
  • CTRL-G 用于显示当前光标所在位置和文件状态信息

# edit

在 Vim 中, 要从命令模式进入编辑模式, 你可以按下 i 键(小写字母 "i")或其他一些按键来切换到插入模式, 具体取决于你想要插入文本的位置和模式。以下是一些常见的进入编辑模式的方式:

常用的为: i,I,a,A,o,O,s,S

  • i: 这是最常见的方式, 它允许你在当前光标位置之前插入文本。只需按下 i 键, 然后你可以开始键入文本。
  • I: 与 i 键不同, 按下 I 键会将光标移动到当前行的行首, 然后进入插入模式, 从行首开始插入文本。
  • a: 这个键将在当前光标位置之后进入插入模式, 允许你在光标的右侧插入文本。
  • A: 与 a 键相反, 按下 A 键会将光标移到当前行的行尾, 然后进入插入模式, 从行尾开始插入文本。
  • o: 这个键会在当前行的下面插入一个新的空白行, 并进入插入模式, 允许你在新行上插入文本。
  • O: 与 o 键相反, 按下 O 键会在当前行的上面插入一个新的空白行, 并进入插入模式, 允许你在新行上插入文本。
  • s: 删除光标所在处的字符然后插入需要录入的文本
  • S: 删除光标所在行, 在当前行的行首开始插入需要录入的文本
  • x: 正常模式下, 删除光标后面一个字符
  • cw: 删除从光标处开始到该单词结束的所有字符

  • v: 字符可视化
  • V: 行可视化
  • Ctrl+v: 块状可视化

# cursor

移动光标. link (opens new window)

  • h, j, k, l, 左下上右
  • 0: 移动到行头
  • g0: 移到光标所在屏幕行行首。
  • ^: 移动到本行的第一个不是 blank 字符
  • g^: 但是移动到当前屏幕行第一个非空字符处
  • $: 移动到行尾
  • g$: 移动光标所在屏幕行行尾。
  • f<c>: 移动到本行下一个为 <c> 的字符处
  • nf<c>: 移动到本行光标处开始的第 n 个 字符为 <c> 的地方(n 是 1，2，3，4 ... 数字)
  • Fa: 同 fa 一样，光标移动方向同 fa 相反
  • nFa: 同 nfa 类似，光标移动方向同 nfa 相反
  • nG: 光标定位到第 n 行的行首
  • gg: 光标定位到第一行的行首
  • G: 光标定位到最后一行的行首
  • H: 光标定位到当前屏幕的第一行行首
  • M: 光标移动到当前屏幕的中间
  • L: 光标移动到当前屏幕的尾部
  • zt: 把当前行移动到当前屏幕的最上方，也就是第一行
  • zz: 把当前行移动到当前屏幕的中间
  • zb: 把当前行移动到当前屏幕的尾部
  • w: 前移一个单词，光标停在下一个单词开头
  • W: 移动下一个单词开头，但忽略一些标点
  • e: 前移一个单词，光标停在下一个单词末尾
  • E: 移动到下一个单词末尾，如果词尾有标点，则移动到标点
  • b: 后移一个单词，光标停在上一个单词开头
  • B: 移动到上一个单词开头，忽略一些标点
  • gj: 移动到一段内的下一行
  • gk: 移动到一段内的上一行
  • +/Enter: 把光标移至下一行第一个非空白字符
  • -: 把光标移至上一行第一个非空白字符
  • * 和 # 匹配光标当前所在的单词，移动光标到下一个(或者上一个)匹配的单词( * 是下一个，# 是上一个)
  • %: 匹配括号移动，包括 ( , { , [ 需要把光标先移动到括号上

# search

  • /{目标字符串}: 当前所编辑的文档中正向查找该字符串. 搜索下一个位置只需要按 n 键。要向相反方向查找同上一次的字符串, 请输入大写 N 即可
  • ?{目标字符串} 当前所编辑的文档中反向查找该字符串
  • /\<search_string>/>: < 表示字符串开始, /> 表示字符串结束, 中间的就是需要搜索的字符串表达式
  • :%s/\<word\>/&/gn: 查询匹配的次数 https://vimhelp.org/vim_faq.txt.html#faq-11.1 (opens new window)
  • :%s/\a\+/&/gn: To count the number of alphabetic words in a file
  • :%s/\S\+/&/gn: To count the number of words made up of non-space characters
  • %: 输入 % 可以查找配对的括号 )、]、}

# replace

  • :s/\$1/\$2: 从光标处开始查找替换, 仅仅替换第一次匹配 $1 的地方为 $2
  • :%s/\$1/\$2: 替换掉文件中所有行第一次出现 $1 的地方为 $2
    • :%s/^\s*//g: 删除每一行的行首的空字符串
  • :%s/\$1/\$2/i: i 表示大小写不敏感查找, I 表示大小写敏感
  • :%s/\$1/\$2/gi: 替换掉所有行出现 $1 (不区分大小写) 为 $2
  • :%s/old/new/gc: 替换掉所有行出现 old (不区分大小写) 为 new, 需要确认
  • :.,+2s/foo/bar/g: 当前行 . 与接下来两行 +2
  • :#,#s/old/new/g: 其中 #,# 代表的是替换操作的若干行中首尾两行的行号

# delete

  • dw: 删除一个单词
  • dnw: 删除 n 个单词
  • dd: 删除一整行
  • ndd: 删除光标处开始的 n 行
  • d$: 删除光标到本行的结尾
  • 1,2d: 删除 1 到 2 的行
  • :5,$d: 删除第五行到结尾的所有行
  • df<c>: 删除光标处到下一个 <c> 的字符处
  • dfn<c>: 删除光标处到第 n 个 <c> 的字符处
  • dH: 删除屏幕显示的第一行文本到光标所在的行
  • dG: 删除光标所在行到文本的结束
  • :%d: 删除文件的所有内容

  1. 复制多行

复制 100 行, 比如从第一行复制到 100 行

一、 复制到 vim 寄存器

在 Vim 中复制多行文本, 可以通过以下步骤实现:

  • 进入普通模式(Normal Mode): 按下 Esc 键, 确保你在普通模式下。

  • 移动光标到第一行: 可以使用 gg 命令将光标移动到文件的第一行。

  • 开始选择文本: 按下 V 键(注意是大写的 V), 进入可视行模式(Visual Line Mode)。这会选中当前光标所在的整行。

  • 移动光标并选择多行: 按下 100G 或者 100gg 命令, 将光标移动到第 100 行。这样, Vim 将会选中从第一行到第 100 行的所有内容。

  • 复制选中的文本: 按下 y 键, 将选中的文本复制到 Vim 的寄存器中。

  • 粘贴文本: 移动光标到你想要粘贴的位置, 然后按下 p 键, 在光标后粘贴文本；或者按下 P 键, 在光标前粘贴文本。

总结: 按下 V 进入可视行模式, 移动光标到第 100 行, 按下 y 复制选中的文本, 按下 p 或 P 粘贴文本。

二、复制到 剪切板

如果你想把复制的内容放到剪切板, 可以这样做。 我的思路是直接保存到文件中

  • 打开其他文件: 使用 :e <文件名> 命令打开你想要粘贴内容的其他文件。
  • 移动光标到你想要粘贴的位置。
  • 粘贴文本: 按下 p, 在光标后粘贴文本；或按下 P, 在光标前粘贴文本。

总结: 在普通模式下, 使用 :e <文件名> 打开其他文件, 移动光标到粘贴位置, 然后按下 p 或 P, 在光标后或光标前粘贴之前复制的文本。如果需要退出并不保存当前文件, 请按下 :q!。

# exit

  • :w
  • :w!
  • :w \$file
  • :q!
  • :qa!: 退出所有的文件, 对所有的文件修改都不做保存
  • :wq
  • :x: 退出文件并保存对文件的修改
  • :saveas file: 另存为

# other

  • <command> | vim -: 通过管道，将输出用 vim 打开。 告诉 vim 从标准输入(stdin)读取输入，而不是从文件中读取
  • u: 撤销
  • ctrl + r: 恢复撤销操作
  • ctrl + f: 查看下一页内容
  • ctrl + b: 查看上一页内容
  • :e \$file: 打开另一个文件
  • :e!: 放弃对文件的所有修改, 恢复文件到上次保存的位置
  • :!: 然后紧接着输入一个外部命令可以执行该外部命令
  • :help quickfix: quickfix 帮助文档
