提示

vim linux 上使用较多的文本编辑器。 具有程序编辑的能力，可以主动的以字体颜色辨别语法的正确性，方便程序设计

  1. vim 提供了交互式教程工具 vimtutor， 这对熟悉 vim 基础用法非常有用
  2. 另外你可以通过在文件 ~/.vimrc 为 vim 设置配置， 参考链接 (opens new window)

# 基础操作

# 命令模式

在正常模式中，按下:（冒号）键或者 /（斜杠），会进入命令模式。在命令模式中可以执行一些输入并执行一些 VIM 或插件提供的指令，就像在 shell 里一样。这些指令包括设置环境、文件操作、调用某个功能等等。

  • set nu 显示行
  • set nonu
  • set nu!: 在行号与非显示行号切换
  • :n , 定位指定行
  • :set ic , 编辑器将不会区分大小写
  • :set noic

# 编辑模式

在 Vim 中，要从命令模式进入编辑模式，你可以按下 i 键（小写字母 "i"）或其他一些按键来切换到插入模式，具体取决于你想要插入文本的位置和模式。以下是一些常见的进入编辑模式的方式：

常用的为： i,I,a,A,o,O,s,S

  • 按下 i 键: 这是最常见的方式，它允许你在当前光标位置之前插入文本。只需按下 i 键，然后你可以开始键入文本。

  • 按下 I 键: 与 i 键不同，按下 I 键会将光标移动到当前行的行首，然后进入插入模式，从行首开始插入文本。

  • 按下 a 键: 这个键将在当前光标位置之后进入插入模式，允许你在光标的右侧插入文本。

  • 按下 A 键: 与 a 键相反，按下 A 键会将光标移到当前行的行尾，然后进入插入模式，从行尾开始插入文本。

  • 按下 o 键: 这个键会在当前行的下面插入一个新的空白行，并进入插入模式，允许你在新行上插入文本。

  • 按下 O 键: 与 o 键相反，按下 O 键会在当前行的上面插入一个新的空白行，并进入插入模式，允许你在新行上插入文本。

  • s , 删除光标所在处的字符然后插入需要录入的文本

  • S , 删除光标所在行，在当前行的行首开始插入需要录入的文本

  • cw , 删除从光标处开始到该单词结束的所有字符

  • 输入 gg 可以使得当前光标直接跳转到文件第一行。

# 查找

  • /{目标字符串} , 当前所编辑的文档中正向查找该字符串. 搜索下一个位置只需要按 n 键。要向相反方向查找同上一次的字符串，请输入大写 N 即可
  • ?{目标字符串} 当前所编辑的文档中反向查找该字符串
  • /\<search_string>/> , < 表示字符串开始，/> 表示字符串结束，中间的就是需要搜索的字符串表达式
  • :%s/\<word\>/&/gn , 查询匹配的次数 https://vimhelp.org/vim_faq.txt.html#faq-11.1
  • :%s/\a\+/&/gn , To count the number of alphabetic words in a file
  • :%s/\S\+/&/gn , To count the number of words made up of non-space characters
  • % , 输入 % 可以查找配对的括号 )、]、}

# 替换

  • :s/\$1/\$2 , 从光标处开始查找替换，仅仅替换第一次匹配 $1 的地方为 $2
  • :%s/\$1/\$2 , 替换掉文件中所有行第一次出现 $1 的地方为 $2
    • :%s/^\s*//g: 删除每一行的行首的空字符串
  • :%s/\$1/\$2/i , i 表示大小写不敏感查找，I 表示大小写敏感
  • :%s/\$1/\$2/gi , 替换掉所有行出现 $1 (不区分大小写) 为 $2
  • :%s/old/new/gc , 替换掉所有行出现 old (不区分大小写) 为 new, 需要确认
  • :.,+2s/foo/bar/g , 当前行 . 与接下来两行 +2
  • :#,#s/old/new/g , 其中 #,# 代表的是替换操作的若干行中首尾两行的行号

# 复制多行

复制 100 行，比如从第一行复制到 100 行

一、 复制到 vim 寄存器

在 Vim 中复制多行文本，可以通过以下步骤实现：

  • 进入普通模式（Normal Mode）：按下 Esc 键，确保你在普通模式下。

  • 移动光标到第一行：可以使用 gg 命令将光标移动到文件的第一行。

  • 开始选择文本：按下 V 键（注意是大写的 V），进入可视行模式（Visual Line Mode）。这会选中当前光标所在的整行。

  • 移动光标并选择多行：按下 100G 或者 100gg 命令，将光标移动到第 100 行。这样，Vim 将会选中从第一行到第 100 行的所有内容。

  • 复制选中的文本：按下 y 键，将选中的文本复制到 Vim 的寄存器中。

  • 粘贴文本：移动光标到你想要粘贴的位置，然后按下 p 键，在光标后粘贴文本；或者按下 P 键，在光标前粘贴文本。

总结：按下 V 进入可视行模式，移动光标到第 100 行，按下 y 复制选中的文本，按下 p 或 P 粘贴文本。

二、复制到 剪切板

如果你想把复制的内容放到剪切板，可以这样做。 我的思路是直接保存到文件中

  • 打开其他文件：使用 :e <文件名> 命令打开你想要粘贴内容的其他文件。
  • 移动光标到你想要粘贴的位置。
  • 粘贴文本：按下 p 键，在光标后粘贴文本；或按下 P 键，在光标前粘贴文本。

总结：在普通模式下，使用 :e <文件名> 打开其他文件，移动光标到粘贴位置，然后按下 p 或 P 键，在光标后或光标前粘贴之前复制的文本。如果需要退出并不保存当前文件，请按下 :q!。

# vimtutor

vimtutor 是一个内置于 Vim 编辑器中的交互式教程，旨在帮助用户学习和掌握 Vim 编辑器的基本用法和功能。通过运行 vimtutor 命令，您可以在终端中启动这个教程并按照提示进行学习。

以下是使用 vimtutor 的一般步骤：

  • 打开终端（命令行界面）。
  • 输入 vimtutor 命令并按下回车键。
  • Vim 将以教程模式启动，并在终端中显示教程的内容。

教程内容会逐步引导您学习 Vim 的各种基本操作和编辑技巧，包括移动光标、插入和删除文本、保存和退出文件等。您可以按照屏幕上的指示进行练习和操作。教程中的练习文件会被保存在您当前的工作目录中，命名为 tutor。

请注意，vimtutor 教程相当详细，需要耐心和时间来完成。建议您安排足够的时间来完成教程，并根据需要进行实践和反复练习，以便更好地掌握 Vim 编辑器的使用技巧。

# vimgrep

vimgrep 是 Vim 中用于执行 全局搜索的命令，它可以在整个文件或多个文件中搜索指定的模式并将匹配的结果保存在 Vim 的 quickfix 列表中. 使用 vimgrep 命令可以在 Vim 中执行全局搜索并轻松导航匹配的结果。它非常适用于处理包含大量文件的项目。

vim中启动vimgrep流程

  1. 进入 Vim 编辑器，打开一个文件。
  2. 进入命令行模式，按下冒号（:）键。
  3. 输入以下命令格式：:vimgrep /模式/ 文件列表
  • /模式/：指定要搜索的模式，可以是简单的文本或正则表达式。
  • 文件列表：指定要在其中搜索的文件列表, 其中"%"表示当前文件。你可以使用通配符来匹配多个文件，例如 **/*.txt 表示当前目录包括子目录所有以 .txt 结尾的文件。
  1. 按下回车键，Vim 将执行搜索并将匹配的结果保存在 quickfix 列表中。
  2. 要打开 quickfix 窗口并查看搜索结果，可以使用命令 :copen。
  3. 在 quickfix 窗口中，你可以使用 :cn 命令跳转到下一个匹配项，使用 :cp 命令跳转到上一个匹配项，使用 :cc 命令跳转到指定的匹配项，使用 :cnext 和 :cprev 命令进行导航。
  4. 使用 :cfilter {pattern}：使用 {pattern} 进行过滤，只显示匹配 {pattern} 的行。这将更新 quickfix 列表为符合过滤条件的结果。
  5. 若要关闭 quickfix 窗口，可以使用命令 :cclose。

# vimgrepadd

vimgrep 和 vimgrepadd 是 Vim 中用于进行全局搜索的命令，但它们之间存在一些区别。

  • vimgrep 命令执行全局搜索，并将搜索结果替换 quickfix 列表中的内容。这意味着每次执行 vimgrep 命令时，quickfix 列表将被新的搜索结果所覆盖。
  • vimgrepadd 命令也执行全局搜索，但它会将搜索结果追加到 quickfix 列表中，而不是替换原有的内容。这样，你可以在多次搜索之间保留和比较不同的搜索结果。

使用 vimgrepadd 的语法与 vimgrep 相同，例如 :vimgrepadd PddVideoEngine xxx.log。执行该命令后，搜索结果将被添加到 quickfix 列表中。

你可以使用 :copen 命令打开 quickfix 窗口查看所有的搜索结果，并使用 :cn 和 :cp 命令分别跳转到下一个和上一个匹配项。

总结而言，vimgrep 用于替换 quickfix 列表内容，而 vimgrepadd 用于追加到 quickfix 列表中，保留多个搜索结果。你可以根据需要选择适合的命令来处理搜索结果。

# 插件管理器 vim-plug

vim-plug: Vim plugin manager (opens new window)

一、安装 vim-plug

curl -fLo ~/.vim/autoload/plug.vim --create-dirs \
    https://raw.githubusercontent.com/junegunn/vim-plug/master/plug.vim
1
2

【解释】这个命令用于安装 Vim-Plug，这是一个流行的 Vim 插件管理器。以下是命令的解释和作用：

  • curl：这是一个用于在命令行中发出 HTTP 请求的命令行工具。它用于从 GitHub 上下载 Vim-Plug 插件管理器。
  • -fLo：这是 curl 命令的选项之一，它的作用是指定输出文件的路径。在这里，它将文件保存到 ~/.vim/autoload/plug.vim。
  • ~/.vim/autoload/plug.vim：这是要保存下载文件的路径。在这里，它将下载的 plug.vim 文件保存到 Vim 的 autoload 目录中。autoload 目录通常用于存储 Vim 脚本和插件。
  • --create-dirs：这是 curl 命令的选项之一，它的作用是在保存文件之前创建目录。如果指定的目录不存在，它会自动创建 autoload 和 ~/.vim 这两个目录。
  • https://raw.githubusercontent.com/junegunn/vim-plug/master/plug.vim：这是要下载的文件的 URL 地址。它指向 Vim-Plug 插件管理器的 GitHub 存储库中的 plug.vim 文件。

执行这个命令后，curl 将从 GitHub 下载 plug.vim 文件并将其保存到 ~/.vim/autoload/plug.vim。这样，你就可以在 Vim 中使用 Vim-Plug 插件管理器来管理和安装其他插件了。要使用 Vim-Plug 安装插件，你可以将插件列表添加到你的 .vimrc 文件中，并在 Vim 中运行 :PlugInstall 命令来安装它们。

# vim 查看另外的文件

vim 在查看过程中，打开另外一个文件

在 Vim 中，你可以在查看一个文件的同时打开另一个文件。这通常通过以下方法实现：

1.分割窗口：Vim 支持分割窗口的功能，你可以在一个窗口中查看一个文件，同时在另一个窗口中打开并查看另一个文件。以下是一些相关命令：

  • 打开新文件并水平分割窗口：:sp filename

  • 打开新文件并垂直分割窗口：:vsp filename

  • 切换窗口：Ctrl-w Ctrl-w

  • 关闭窗口：:q

    2.标签页：另一种方法是使用标签页来查看多个文件。你可以使用以下命令：

  • 打开新文件并将其放入新标签页：:tabe filename

  • 切换标签页：:tabn (下一个标签页) 或 :tabp (上一个标签页)

  • 关闭标签页：:tabc

    3.缓冲区：Vim 也支持多个缓冲区，你可以使用以下命令在不同的缓冲区之间切换：

  • 打开新文件并加载到缓冲区：:e filename

  • 切换缓冲区：:bnext (下一个缓冲区) 或 :bprev (上一个缓冲区)

  • 关闭缓冲区：:bd

这些方法中的选择取决于你的使用习惯和需求。你可以根据需要在不同的文件之间切换以提高编辑效率。

# 多个 quickFix 窗口

在 Vim 中，你可以打开多个 Quickfix 窗口，但需要借助一些插件或者自定义脚本来实现。Quickfix 窗口通常用于显示编译错误、代码搜索结果或其他类型的快速查找结果。以下是一种实现多个 Quickfix 窗口的方法：

使用插件： 有一些 Vim 插件可以帮助你管理多个 Quickfix 窗口。一个常用的插件是 qfenter (opens new window)，它允许你在 Vim 中打开多个 Quickfix 窗口。你可以按照以下步骤安装它：

  1. 使用 Vim 插件管理器（如 Vim-Plug）安装 qfenter 插件。将以下行添加到你的.vimrc 文件中：
Plug 'yssl/QFEnter.vim'
1

  1. 运行 Vim 并执行:PlugInstall 来安装插件。

  2. 使用:QFEnter 命令来打开新的 Quickfix 窗口。你可以在不同的 Quickfix 窗口中加载不同的 Quickfix 列表。

  • 打开新的 Quickfix 窗口： :QFEnter
  • 切换到不同的 Quickfix 窗口： :QFEnter <number>

这样，你就可以在同一 Vim 会话中管理多个 Quickfix 窗口了。

请注意，Vim 的基本功能中并没有直接支持打开多个 Quickfix 窗口，所以需要使用插件来扩展这个功能。

# visual 模式复制问题

在使用 vim 的时候，如果显示行号，在 visual 模式下，复制可能会有空格

在 Vim 中，在显示行号的情况下，复制文本可能会包括行号后面的空格。这是因为行号是显示在文本前面的，所以在 Visual 模式下选择文本时也会选择行号后的空格。

如果你希望在复制文本时不包括行号后的空格，可以考虑在 Visual 模式下使用一些技巧来排除行号。以下是一些方法：

  1. 使用 set nu!命令切换行号显示：你可以使用命令:set nu!来切换行号的显示。这个命令会在显示和隐藏行号之间切换。在不显示行号的情况下，复制操作不会包括行号后的空格。当你需要查看行号时，再次执行:set nu!即可。

  2. 使用正则表达式来选择文本：在 Visual 模式下，你可以使用正则表达式来选择文本，以便排除行号。例如，假设你的行号是左对齐的，你可以使用正则表达式来选择非空格字符：

  • 进入 Visual 模式：按 V 键。
  • 使用正则表达式选择文本：/\S.*，然后按 Enter。这将选择当前行中第一个非空格字符之后的文本。
  1. 使用 Vim 插件：有一些 Vim 插件可以帮助你更精确地选择文本，以排除行号。例如，"vim-textobj-line" 插件允许你选择文本行，而不包括行号。你可以使用插件管理器来安装这些插件。

根据你的偏好和需求，选择适合你的方法来避免在 Visual 模式下复制行号后的空格。
