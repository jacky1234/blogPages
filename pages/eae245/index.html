# Perfetto

提示

perfetto is a tool that lets you collect performance information from Android devices via the Android Debug Bridge (ADB). Invoke the perfetto tool using the adb shell perfetto ... command. perfetto uses various sources to collect performance traces from your device, such as:

  • ftrace for information from the kernel
  • atrace for user-space annotation in services and apps
  • heapprofd for native memory usage information of services and apps

更多内容参考: android 官网给出的 perfetto command line 介绍 (opens new window)

# 分析卡顿技巧

使用 Perfetto 来分析卡顿问题可以帮助你找出引起应用程序响应缓慢的原因。以下是一些技巧可以帮助你在使用 Perfetto 进行卡顿分析时取得更好的结果：

  1. 确定卡顿发生的时间：观察应用程序的响应缓慢的时间段，并记下具体的时间范围。这将有助于你在 Perfetto 中选择正确的时间段进行分析。
  2. 使用 Perfetto 录制跟踪数据：在 Perfetto UI 中，选择适当的跟踪配置，并开始录制跟踪数据。确保在卡顿发生期间进行录制，以捕获相关的事件和活动。
  3. 分析 CPU 使用情况：在 Perfetto UI 的时间轴视图中，查看 CPU 使用率和 CPU 调度事件。观察是否存在 CPU 占用过高或频繁切换的情况，这可能会导致卡顿。
  4. 检查主线程活动：关注主线程的活动，特别是长时间运行的任务和阻塞操作。查看主线程的事件和函数调用，确定是否有耗时操作或阻塞导致卡顿。
  5. 分析 I/O 操作：观察磁盘和网络 I/O 操作的情况，特别是文件读写和网络请求。长时间的 I/O 操作可能会导致卡顿现象。
  6. 检查 UI 渲染性能：关注 UI 渲染相关的事件和活动，包括布局、绘制和动画。观察是否有大量的 UI 渲染操作或者渲染过程耗时过长。
  7. 使用 Perfetto 内置分析工具：Perfetto 提供了一些内置的分析工具，如耗时图表、火焰图、调用堆栈等，可以帮助你更深入地分析卡顿问题。尝试使用这些工具来定位具体的性能瓶颈。
  8. 结合其他工具和数据：除了 Perfetto，还可以结合其他性能分析工具和数据来进行综合分析。例如，使用 Systrace 来观察系统级别的事件，使用应用程序级别的监测工具来获取更详细的应用程序行为数据。

以上是使用 Perfetto 进行卡顿分析的一些建议和技巧。请记住，分析卡顿问题可能需要结合应用程序本身的特点和具体情况来选择合适的分析方法和工具。
