# useEffect

useEffect (opens new window) is a React Hook that lets you synchronize a component with an external system.

import { useEffect } from 'react';
import { createConnection } from './chat.js';

function ChatRoom({ roomId }) {
  const [serverUrl, setServerUrl] = useState('https://localhost:1234');

  useEffect(() => {
    const connection = createConnection(serverUrl, roomId);
    connection.connect();
    return () => {
      connection.disconnect();
    };
  }, [serverUrl, roomId]);
  // ...
}

# useImperativeHandle

useImperativeHandle (opens new window) is a React Hook that lets you customize the handle exposed as a ref.

**在函数组件中使用 forwardRef 和 useImperativeHandle 的一般步骤如下: **

  • 使用 forwardRef 包裹你的组件函数, 以便能够接收 ref 参数。
  • 在组件函数中定义内部的 handle 对象, 它包含你想要暴露给父组件的方法和属性。
  • 使用 useImperativeHandle 钩子, 将 ref 和回调函数作为参数传递给它。回调函数应返回你定义的 handle 对象。
  • 如果有必要, 可以在第三个参数中传递依赖项数组, 以在依赖项发生变化时重新生成 handle 对象

# case 1

import * as React from 'react'

const CustomComponent = React.forwardRef((props, ref) => {
  // Declare your handle object with desired methods
  const handle = {
    // Custom methods and properties
  };

  // Expose the handle to the parent component
  React.useImperativeHandle(ref, () => {
    return {
      xx: (a: number) => {

      },
    }
  }, [/* dependencies */]);

  // Rest of your component code
  return (
    // JSX for your component
  );
});

export default CustomComponent;

参数: dependencies

optional dependencies: The list of all reactive values referenced inside of the createHandle code. Reactive values include props, state, and all the variables and functions declared directly inside your component body. If your linter is configured for React, it will verify that every reactive value is correctly specified as a dependency. The list of dependencies must have a constant number of items and be written inline like [dep1, dep2, dep3]. React will compare each dependency with its previous value using the Object.is comparison. If a re-render resulted in a change to some dependency, or if you omitted this argument, your createHandle function will re-execute, and the newly created handle will be assigned to the ref.

在上述示例中, 我们使用 forwardRef 将组件包装起来, 以便能够接收 ref 属性。然后, 我们在组件的顶层使用 useImperativeHandle, 将自定义的 handle 对象作为第一个参数传递给该钩子函数。通过这样做, 我们可以自定义暴露给父组件的引用对象。

确保将自定义的方法和属性添加到 handle 对象中, 并根据需要调整 useImperativeHandle 中的依赖项列表。这样, 在使用组件时, 父组件就可以通过引用对象ref.current调用自定义的方法和访问属性.

# useMemo

useMemo (opens new window) is a React Hook that lets you cache the result of a calculation between re-renders.

// snip1: not
const visibleTodos = useMemo(() => {
  // 🚩 Mistake: mutating a prop
  todos.push({ id: 'last', text: 'Go for a walk!' });
  const filtered = filterTodos(todos, tab);
  return filtered;
}, [todos, tab]);

// snip2: use this
const visibleTodos = useMemo(() => {
  const filtered = filterTodos(todos, tab);
  // ✅ Correct: mutating an object you created during the calculation
  filtered.push({ id: 'last', text: 'Go for a walk!' });
  return filtered;
}, [todos, tab]);

这段文档在说明一个常见的错误和正确的做法, 涉及到在 React 中使用 useMemo 钩子来进行性能优化时的注意事项。

在第一个示例中, 开发者试图使用 useMemo 钩子来计算 visibleTodos, 但在计算过程中直接对传入的 todos 数组进行了修改, 向其添加了一个新的 todo。这是一个错误的做法, 因为 useMemo 钩子可能会在同一个计算周期内多次调用你的函数, 从而导致相同的 todo 被添加多次。这种错误会导致应用程序状态的不一致性和预期之外的行为。

为了解决这个问题, 第二个示例提供了一个正确的做法。在这个示例中, 开发者避免直接修改传入的 todos 数组, 而是在计算过程中创建了一个新的数组 filtered, 并在计算完成后对新数组进行了修改。这样做是安全的, 因为它不会影响传入的 todos 数组, 而只会修改在计算过程中创建的新数组。

综上所述, 文档提醒开发者在使用 useMemo 钩子时要注意不要直接修改传入的 props, 而是应该创建新的对象或数组, 并仅对新对象或数组进行修改。这样可以确保应用程序的状态一致性, 并避免意外的副作用。
